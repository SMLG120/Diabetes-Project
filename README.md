## 📌 Project Overview

Binary classification pipeline to predict the **probability of diabetes** from tabular patient health data. Trained and evaluated on Kaggle's Playground Series S5E12 dataset.

**Primary metric:** ROC-AUC

## 🗂️ Dataset Description

| Field | Description |
|---|---|
| Source | Kaggle Playground Series S5E12 (synthetically generated) |
| Target | `diabetes` — 1 = has diabetes, 0 = no diabetes |
| Features | glucose, bmi, age, blood_pressure, insulin, skin_thickness, pregnancies, dpf |
| Train size | ~100,000 rows |
| Test size | ~66,800 rows |

---

## 🔬 Methodology

### 1. EDA (Member 1)
- Feature distributions, boxplots, correlation heatmap
- Class imbalance check, outlier detection

### 2. Preprocessing (Member 1)
- Median/mode imputation for missing values
- Interaction feature engineering (bmi×age, glucose/insulin, etc.)
- StandardScaler (fit on train only — no leakage)

### 3. Models (Member 2)
| Model | Type |
|---|---|
| Logistic Regression | Baseline (linear) |
| Random Forest | Intermediate (bagging) |
| LightGBM | Advanced (boosting) |
| XGBoost | Advanced (boosting) |
| CatBoost | Advanced (boosting) |

All trained with **Stratified 5-Fold Cross-Validation** and **class imbalance handling**.

Optional: Optuna hyperparameter tuning (`--tune` flag).

### 4. Ensemble (Member 3)
- Simple averaging
- Top-k model averaging
- Weighted blending (weights ∝ CV AUC score)

## 🚀 Quick Start

### 1. Install Dependencies
```bash
pip install -r requirements.txt
```

### 2. Get the Data
Download `train.csv` and `test.csv` from:  
👉 https://www.kaggle.com/competitions/playground-series-s5e12/data  
Place them in the `data/` folder.

### 3. Run the Full Pipeline
```bash
# Standard run
python run_pipeline.py

# With Optuna hyperparameter tuning (recommended for best score)
python run_pipeline.py --tune --n_trials 100

# After getting your Kaggle leaderboard score
python run_pipeline.py --lb 0.8812
```

### 4. Individual Scripts
```bash
# EDA only
python notebooks/eda.py

# Preprocessing only
python src/preprocessing.py

# Training only
python src/model_training.py

# Evaluation only
python src/evaluation.py
```

```python

np.random.seed(RANDOM_STATE)
```

---

## 📝 Notes on Overfitting

- Monitor CV ↔ Leaderboard gap (target: < 0.005)
- Regularize boosting models: `reg_alpha`, `reg_lambda`, `min_child_samples`
- Never fit scaler/imputer on full dataset before CV splits
